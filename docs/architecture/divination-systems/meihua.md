# 梅花易数系统说明

**系统类型**：`DivinationType.meiHua`  
**状态**：Disabled / Core Cast Implemented  
**当前权威实现范围**：

- `lib/divination_systems/meihua/meihua_system.dart`
- `lib/divination_systems/meihua/models/meihua_result.dart`

---

## 1. 文档定位

梅花易数当前尚未启用 UI，但核心排盘模型与主算法已经落地。

因此本文件既是“当前实现说明”，也是**启用前必须满足的正式契约**。  
后续梅花系统的起卦方式、结果模型、结果页、历史卡片、AI 结构化输出，都必须以本文件为准。

---

## 2. 系统定位

梅花易数第一版的定位必须明确：

1. 它不是六爻的简化版。
2. 它不是以纳甲、六亲、六神、世应为中心的系统。
3. 它的核心是：
   - 起卦依据
   - 本卦
   - 动爻
   - 变卦
   - 互卦
   - 体卦 / 用卦
   - 体用五行关系

第一版产品形态应采用**轻盘面、重推演链**。

这意味着：

- 重点展示“这个卦是怎么来的”
- 重点展示“为什么判体用”
- 重点展示体用五行关系
- 不要把结果页做成另一张六爻大表

占断不在排盘结果里给结论，占断由 AI 分析负责。

---

## 3. 当前代码状态

当前实现事实如下：

- `isEnabled = false`
- `cast()` 已实现 `time / number / manual`
- `validateInput()` 已按三种方式严格校验
- `MeiHuaResult` 已替换为正式结果模型
- `getSummary()` 已返回真实业务摘要
- 没有 UI 工厂实现
- 没有 AI 结构化 formatter

代码注释里历史上提到过“物象起卦”，但当前 `supportedMethods` 并未正式开放该方式，因此它不是当前对外契约的一部分。

## 4. 第一版边界

梅花易数第一版必须主动控制范围，避免一开始就把规则做散。

### 4.1 第一版必须支持

| 方法 | `CastMethod` | 状态 | 说明 |
|---|---|---|---|
| 时间起卦 | `time` | 必须支持 | 基于起卦时间推上卦、下卦、动爻 |
| 数字起卦 | `number` | 必须支持 | 输入两个数字推上下卦，合数推动爻 |
| 手动输入 | `manual` | 必须支持 | 直接指定上卦、下卦、动爻 |

### 4.2 第一版明确不做

1. 不做物象起卦
2. 不做报数卦单独入口
3. 不做电脑随机起卦
4. 不做纳甲六亲
5. 不做六神
6. 不做世应
7. 不做错卦、综卦单独展示
8. 不做多动爻
9. 不做多流派兼容

### 4.3 第一版单一规则

第一版必须固定一条规则链，不允许“先做两种说法再兼容”。

尤其以下三点必须写死：

1. 时间起卦到底使用什么数字源
2. 动爻如何确定
3. 体卦 / 用卦如何确定

## 5. 基础映射规则

### 5.1 八卦数字映射

梅花易数第一版固定采用以下八卦序数：

| 数字 | 卦 | 象 | 五行 |
|---|---|---|---|
| 1 | 乾 | 天 | 金 |
| 2 | 兑 | 泽 | 金 |
| 3 | 离 | 火 | 火 |
| 4 | 震 | 雷 | 木 |
| 5 | 巽 | 风 | 木 |
| 6 | 坎 | 水 | 水 |
| 7 | 艮 | 山 | 土 |
| 8 | 坤 | 地 | 土 |

取模规则固定为：

- `% 8 == 0` 时按 `8` 处理，即落 `坤`

### 5.2 地支序数映射

时间起卦里的年支与时支，一律采用以下序数：

| 地支 | 数 |
|---|---|
| 子 | 1 |
| 丑 | 2 |
| 寅 | 3 |
| 卯 | 4 |
| 辰 | 5 |
| 巳 | 6 |
| 午 | 7 |
| 未 | 8 |
| 申 | 9 |
| 酉 | 10 |
| 戌 | 11 |
| 亥 | 12 |

### 5.3 动爻序数

梅花第一版的动爻必须是**单动爻**，序号固定如下：

| 数字 | 含义 |
|---|---|
| 1 | 初爻 |
| 2 | 二爻 |
| 3 | 三爻 |
| 4 | 四爻 |
| 5 | 五爻 |
| 6 | 上爻 |

规则固定为：

- `% 6 == 0` 时按 `6` 处理，即落上爻

## 6. 起卦方式与输入契约

### 6.1 `time`

```dart
{}
```

说明：

- 依赖 `castTime`
- 结果页展示时间时使用阳历 + 农历
- 取数时使用农历上下文，不使用公历年月日直接取数

### 时间起卦固定算法

时间起卦第一版固定采用：

1. 年数 = **农历年支数**
2. 月数 = **农历月数**
3. 日数 = **农历日数**
4. 时数 = **时支数**

进一步固定为：

- 上卦数 = `(年数 + 月数 + 日数) % 8`
- 下卦数 = `(年数 + 月数 + 日数 + 时数) % 8`
- 动爻数 = `(年数 + 月数 + 日数 + 时数) % 6`

补充规则：

- 上卦数若为 `0`，按 `8` 处理
- 下卦数若为 `0`，按 `8` 处理
- 动爻数若为 `0`，按 `6` 处理

### 时间起卦实现要求

结果对象必须保留以下取数痕迹：

- 年支
- 年支数
- 月数
- 日数
- 时支
- 时支数
- 上卦数
- 下卦数
- 动爻数

这些不是调试信息，而是正式结果的一部分。

### 6.2 `number`

```dart
{
  'upperNumber': 12,
  'lowerNumber': 8,
}
```

约束：

- 两个字段都必须为 `int`
- 两个字段都必须大于 `0`

### 数字起卦固定算法

- 上卦数 = `upperNumber % 8`
- 下卦数 = `lowerNumber % 8`
- 动爻数 = `(upperNumber + lowerNumber) % 6`

补充规则：

- 上卦数若为 `0`，按 `8` 处理
- 下卦数若为 `0`，按 `8` 处理
- 动爻数若为 `0`，按 `6` 处理

### 不采用单一 `number` 字段

第一版不接受：

```dart
{
  'number': 123
}
```

原因不是“不能算”，而是协议太含糊：

- 无法知道上卦、下卦来源
- 无法清晰追溯动爻来源
- 不利于结果页展示“起卦依据”

### 6.3 `manual`

```dart
{
  'upperTrigram': '乾',
  'lowerTrigram': '巽',
  'movingLine': 3,
}
```

约束：

- `upperTrigram` 必须为合法八卦
- `lowerTrigram` 必须为合法八卦
- `movingLine` 必须在 `1..6`

### 手动输入说明

手动模式下：

- 起卦时间仍然需要保留在结果对象中，用于历史记录和时间展示
- 但本卦、动爻的生成以手动输入为准，不再回头参与时间取数

### 6.4 `validateInput()` 目标行为

梅花易数启用前，`validateInput()` 必须严格收敛为：

- `time`：始终通过
- `number`：只接受 `upperNumber` + `lowerNumber`
- `manual`：只接受 `upperTrigram` + `lowerTrigram` + `movingLine`

不得：

- 无条件返回 `true`
- 自动吞掉缺字段输入
- 静默回退默认卦

## 7. 排盘主链

梅花易数第一版的排盘链路必须固定为：

1. 统一输入源，得到 `上卦 / 下卦 / 动爻`
2. 生成本卦
3. 依据单动爻生成变卦
4. 依据本卦生成互卦
5. 依据动爻所在卦确定体卦 / 用卦
6. 分析体卦与用卦五行关系
7. 输出结构化排盘与摘要（占断不在此链内，交由 AI 负责）

### 7.1 本卦

本卦由：

- 上卦
- 下卦

组合而成。

结果对象中必须能同时访问：

- 上卦名
- 下卦名
- 本卦名
- 六爻阴阳结构

### 7.2 变卦

变卦必须由**本卦 + 单动爻翻转**推得。

规则：

- 阳爻变阴
- 阴爻变阳

### 7.3 互卦

互卦固定由本卦中间四爻推出：

- 2、3、4 爻组成互卦下卦
- 3、4、5 爻组成互卦上卦

这一步必须是正式字段，不得只在解释里临时生成。

### 7.4 体卦 / 用卦

第一版固定采用以下规则：

- 动爻落在 `1..3`：**下卦为用，上卦为体**
- 动爻落在 `4..6`：**上卦为用，下卦为体**

这条规则必须直接写进结果对象或结果元数据中，不能只靠 UI 文案猜。

### 7.5 五行关系

体用关系只分析**体卦五行 vs 用卦五行**。

结果必须归一到以下五类之一：

- 体生用
- 用生体
- 体克用
- 用克体
- 体用比和

### 7.6 占断边界

第一版排盘**不生成**占断结论，也不落下 `judgement` / `detail` 之类的断语字段。

- 排盘层只产出可验证的结构化字段（本/变/互/动爻/体/用/五行关系）
- `getSummary()` 只基于上述结构化字段拼一行文本摘要
- 所有对事情吉凶、走向、建议的解读一律走 AI 分析
- 原因是断语属于高度主观、流派分歧大的输出，不应写死在排盘引擎里

## 11. AI 结构化输出规范

梅花易数启用时，必须同步提供结构化 formatter。

标题固定为：

```text
【梅花易数完整结构化排盘】
```

区块顺序固定为：

1. 排盘总览
2. 起卦依据
3. 卦象结构
4. 体用与五行

占断结论不在结构化输出里给出——AI 会基于以上四块独立生成断语。

### 11.1 目标格式模板

```text
【梅花易数完整结构化排盘】

一、排盘总览
- 起卦：2026-04-19 09:22（农历三月初三）
- 方式：时间起卦
- 四柱：丙午年 壬辰月 癸亥日 丁巳时
- 本卦：<本卦名>
- 变卦：<变卦名>
- 互卦：<互卦名>
- 动爻：<初爻/二爻/.../上爻>
- 体卦：<体卦名>
- 用卦：<用卦名>
- 体用关系：<体生用/用生体/体克用/用克体/体用比和>

二、起卦依据
- 规则：年支数 + 月数 + 日数取上卦；年支数 + 月数 + 日数 + 时支数取下卦与动爻
- 年支：<年支>，数<年支数>
- 月数：<月数>
- 日数：<日数>
- 时支：<时支>，数<时支数>
- 上卦数：<上卦数> -> <上卦>
- 下卦数：<下卦数> -> <下卦>
- 动爻数：<动爻数> -> <动爻名称>

三、卦象结构
- 本卦：上<上卦>下<下卦>，<本卦名>
- 变卦：上<变卦上卦>下<变卦下卦>，<变卦名>
- 互卦：上<互卦上卦>下<互卦下卦>，<互卦名>

四、体用与五行
- 动爻落<上卦/下卦>，故<哪一卦为体，哪一卦为用>
- 体卦：<体卦名>，<五行>
- 用卦：<用卦名>，<五行>
- 关系：<体用五行关系>
```

### 11.2 四条硬规则

1. `起卦依据` 必须输出，不能只给结果不给来源。
2. `本卦 / 变卦 / 互卦 / 动爻 / 体卦 / 用卦 / 体用关系` 必须同时出现。
3. 第一版结构化输出不引入纳甲、六亲、六神、世应字段。
4. 第一版结构化输出**不包含占断结论段**，占断留给 AI 根据结构化输出独立生成。

### 11.3 固定校盘样例

以下样例可直接用于开发联调与回归测试：

```text
起卦时间：2026-04-19 09:22
农历：三月初三
年支：午，数 7
月数：3
日数：3
时支：巳，数 6

上卦原数：7 + 3 + 3 = 13，13 % 8 = 5 -> 巽
下卦原数：7 + 3 + 3 + 6 = 19，19 % 8 = 3 -> 离
动爻原数：19，19 % 6 = 1 -> 初爻

本卦：风火家人
变卦：风山渐
互卦：火水未济
体卦：巽
用卦：离
体用关系：体生用
摘要：风火家人 → 风山渐 · 体生用
```

注意：此样例**不**包含占断一句，占断属于 AI 分析的输出，不是排盘的输出。
