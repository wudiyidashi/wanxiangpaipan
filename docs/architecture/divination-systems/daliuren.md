# 大六壬系统说明

**系统类型**：`DivinationType.daLiuRen`  
**状态**：Enabled  
**当前权威实现**：

- `lib/divination_systems/daliuren/daliuren_system.dart`
- `lib/divination_systems/daliuren/models/daliuren_result.dart`
- `lib/divination_systems/daliuren/ui/daliuren_ui_factory.dart`

---

## 1. 系统定位

大六壬系统围绕四课、三传、天盘、十二神将、神煞进行排盘。

它是当前第二个已启用系统，重点不是“像六爻一样展示一张卦表”，而是把静态盘面信息和动态推演信息分层展示。

---

## 2. 支持的排盘方法

| 方法 | `CastMethod` | 状态 | 说明 |
|---|---|---|---|
| 正时起课 | `time` | 已实现 | 使用当前时间或传入的 `castTime` |
| 报数起课 | `reportNumber` | 已实现 | 输入一个整数，映射时支 |
| 指定干支 | `manual` | 已实现 | 指定日干、日支、时支、月建 |
| 随机起课 | `computer` | 已实现 | 随机取地支作为时支 |

---

## 3. 输入契约

### 3.1 通用要求

- `castTime` 可为空；为空时使用当前时间。
- `manual` 的默认值仅是当前代码中的开发期兜底，不是正式契约。
- 一旦来自 UI、接口层或脚本层的调用进入正式流程，`manual` 必须传完整字段。

### 3.2 各方法 payload

#### `time`

```dart
{}
```

说明：

- 不需要额外输入
- 时支来自当前占时

#### `reportNumber`

```dart
{
  'number': 7,
}
```

约束：

- `number` 必须为 `int`
- 系统按 `abs(number) - 1` 对 12 取模映射地支

#### `manual`

```dart
{
  'riGan': '甲',
  'riZhi': '子',
  'shiZhi': '午',
  'yueJian': '寅',
}
```

约束：

- `riGan` 必填，且必须是合法天干
- `riZhi` 必填，且必须是合法地支
- `shiZhi` 必填，且必须是合法地支
- `yueJian` 必填，且必须是合法地支

说明：

- 当前 `validateInput()` 只强制 `riGan`、`riZhi` 非空，`shiZhi` 与 `yueJian` 可空
- 当前 `_castByManual()` 为空时会回退到 `甲 / 子 / 子 / 寅`
- 这套默认值不能视为业务协议，只能视为当前实现缺陷

#### `computer`

```dart
{}
```

说明：

- 系统随机选择一个地支作为时支
- 之后复用时间起课链路

---

## 4. 排盘计算流程

### 4.1 正时 / 报数 / 随机的主链路

1. 获取农历信息
2. 确定时支
3. 计算天盘
4. 配置十二神将
5. 排四课
6. 推三传
7. 计算神煞
8. 组装 `DaLiuRenResult`

### 4.2 报数与随机是时间起课的变体

- `reportNumber` 只是更换时支来源
- `computer` 只是随机生成时支
- 它们不应派生出独立的结果结构

### 4.3 手动起课的本质

手动起课并不是“跳过算法”，而是手动指定核心上下文后仍走同一套盘面推导。

## 6. 结果页显示规范

### 6.1 区块顺序

大六壬结果页必须按以下顺序展示：

1. 扩展信息
2. 四课
3. 三传
4. 天盘
5. 十二神将
6. 神煞
7. AI 分析

### 6.2 各区块必须显示的要素

#### 扩展信息

- 阳历时间
- 农历月日
- 年月日时干支
- 空亡
- 月建、日建

#### 四课

必须显示：

- 标题：四课
- 传统顺序四列
- 每课的上神
- 每课的下神
- 每课的五行关系

说明：

- 当前 UI 采用 2x2 传统表格展开为四列
- 当前颜色语义已区分贼课、比用、普通状态

#### 三传

必须显示：

- 初传、中传、末传
- 每传的地支
- 每传的六亲
- 课体 badge
- 课体解释文案（若可得）

#### 天盘

当前 P0 必须显示：

- 月将
- 月将名称
- 月将描述

说明：

- 完整 12 位盘面分布属于增强项
- 在增强项落地前，不允许删掉当前最小天盘信息

#### 十二神将

当前 P0 必须显示：

- 贵人所在位置
- 贵人类型描述
- 布神方向说明

#### 神煞

必须分组显示：

- 吉神
- 凶神

每个条目至少显示 `displayText`。

#### AI 分析

- AI 分析位于结构化结果之后
- 不能取代四课、三传、神煞的结构化展示

---

## 7. 历史卡片规范

大六壬历史卡片必须采用 5 层结构：

1. 占问事项
2. 时间
3. 结果摘要
4. 系统类型 badge
5. 起课方式 badge

摘要格式要求：

- 最低要求：`贼克课 · 初传子`
- 当前卡片实现：`贼克课 · 初传子 中传卯 末传申`

推荐保留当前较完整格式，因为它比最低摘要更有检索价值。

---

## 8. 当前实现缺陷与约束

以下问题必须在文档中明确，而不是放任其继续扩散：

1. `manual` 当前仍保留默认值回退，这不是正式业务协议。
2. 天盘结果页目前只展示月将摘要，而不是完整盘面。
3. 起课页当前直接 new `DaLiuRenSystem()` 并自行保存记录，这种流程与六爻的 ViewModel 路径不完全一致，后续若收敛交互层，需要以本文件定义的 I/O 和展示契约为准，不以当前调用路径为准。

---

## 9. 后续改动准则

后续如果改大六壬系统，必须同步修改本文件的场景：

- 调整起课方法
- 修改 `manual` 输入字段
- 调整 `DaLiuRenResult` 字段
- 新增天盘完整盘面展示
- 修改历史摘要格式
