## 1. 文档定位

这份文档不是“大六壬是什么”的介绍文，也不是一次性的界面草稿。

这份文档是大六壬系统的**正式开发依据**，明确回答 7 件事：

1. 大六壬有哪些起课方式
2. 参数面板上每个选项的古法含义是什么
3. 每个参数改动后会影响盘里的哪些结构
4. 正统排盘必须按什么顺序计算
5. 固定样例应该怎样校盘
6. 结果页和历史页必须显示哪些结构化要素
7. 当前代码与规范偏差在哪里，后续必须怎样回收

后续只要改动以下任一项，就必须同步修改本文档：

- 起课方式
- 参数面板字段
- 月将计算规则

- 贵人口诀版本

## 2. 先分清四层概念

大六壬开发最容易失控，是因为把四层东西混成了一层。

### 2.1 起课方式

这是 `CastMethod` 层的入口能力，决定“源头输入是什么”。

当前系统对应：

- `time`
- `reportNumber`
- `manual`
- `computer`

### 2.2 参数面板

这是正统大六壬界面必须暴露的规则参数，决定“同一组输入用哪套古法规则起盘”。

典型参数包括：

- 月将自动 / 手动
- 昼夜自动 / 昼贵 / 夜贵
- 贵人口诀版本
- 日柱旬遁干 / 时柱旬遁干
- 三传显示在上

### 2.3 排盘引擎

这是算法主链，决定“盘是怎么算出来的”。

至少必须按以下顺序执行：

1. 取四柱
2. 定旬空
3. 定月将
4. 定昼夜
5. 定贵人
6. 月将加时起天地盘
7. 排十二天将
8. 排四课
9. 取三传
10. 叠加六亲、遁干、神煞、空亡等辅助信息

### 2.4 结果展示

这是结果页与历史页如何把结构显示出来。

它决定：

- 三传放哪里
- 四课按什么顺序显示
- 天地盘是否完整
- 十二天将是否完整
- 历史卡片摘要怎么写

如果不把这四层拆开，后续开发会出现三类错误：

1. 把界面开关写成服务内常量
2. 把显示序误当成古法课序
3. 把当前错误输出当成“兼容逻辑”固化

## 3. 起课方式与输入契约

本节定义的是**大六壬系统对外输入协议**。

当前代码里仍有一部分简化字段和调试型默认值；这些不是长期契约。长期契约以本节为准，代码必须逐步收敛。

### 3.1 支持的起课方式

| 方法 | `CastMethod` | 当前状态 | 说明 |
|---|---|---|---|
| 时间起课 | `time` | 已实现 | 输入公历时间，自动推出四柱后起课 |
| 报数起课 | `reportNumber` | 已实现但简化 | 当前实现是“一个数映射时支后复用时间起课” |
| 干支起课 | `manual` | 已实现但不完整 | 当前实现更像简化上下文输入，不是完整传统干支起课面板 |
| 电脑起课 | `computer` | 已实现 | 当前实现是随机确定起课上下文后复用时间起课 |

### 3.2 参数面板统一结构

大六壬所有起课方式，最终都要收敛到一份显式参数对象。

推荐目标 payload：

```dart
{
  'params': {
    'birthYear': 1990,
    'monthGeneralMode': 'auto', // auto | manual
    'manualMonthGeneral': '戌',
    'dayNightMode': 'auto', // auto | day | night
    'guiRenVerse': 'classic', // classic | jiaDayAlt
    'xunShouMode': 'day', // day | hour
    'showSanChuanOnTop': true,
  }
}
```

字段说明：

- `birthYear`：本命占扩展参数，时事占可空
- `monthGeneralMode`：月将自动或手动
- `manualMonthGeneral`：手动月将，只有 `monthGeneralMode = manual` 时有效
- `dayNightMode`：昼夜自动或强制昼 / 夜
- `guiRenVerse`：贵人口诀版本
- `xunShouMode`：旬位主轴，`day` 表示日柱旬遁干，`hour` 表示时柱旬遁干
- `showSanChuanOnTop`：纯显示开关

### 3.3 `time`

```dart
{
  'params': {
    ...
  }
}
```

说明：

- 核心输入来自 `castTime`
- 若 `params` 省略，则使用默认正统参数

### 3.4 `reportNumber`

```dart
{
  'number': 7,
  'params': {
    ...
  }
}
```

约束：

- `number` 必须为 `int`
- `params` 可省略，省略时采用默认正统参数

说明：

- 当前实现是“一个数映射时支”
- 这是项目内当前存在的起课入口，不代表古法只有这一种报数逻辑
- 若后续要扩展更完整的报数规则，必须先改本文档

### 3.5 `manual`

目标输入不应再停留在旧的 `riGan/riZhi/shiZhi/yueJian` 四字段简化模式，而应支持完整干支上下文。

推荐目标 payload：

```dart
{
  'yearGanZhi': '丙午',
  'monthGanZhi': '壬辰',
  'dayGanZhi': '壬戌',
  'hourGanZhi': '辛亥',
  'params': {
    ...
  }
}
```

约束：

- 四柱全部必填
- 干支必须是合法组合
- `params` 可省略，省略时采用默认正统参数

当前代码问题：

- 当前 `manual` 仍允许缺省回退到调试值
- 这对开发方便，但不属于正式契约

后续收敛要求：

- UI 层必须显式传完整四柱
- 服务层不得在正式流程里静默拼默认柱

### 3.6 `computer`

```dart
{
  'params': {
    ...
  }
}
```

说明：

- 不需要额外业务输入
- 系统负责随机生成起课上下文
- 生成后仍必须走同一条正统排盘链

## 4. 参数面板规则

本节定义的是**正统大六壬界面必须具备的参数项**。

这些选项不是“软件花哨设置”，而是会直接改变月将、天盘、神将、四课、三传的底层公式参数。

---

## 4.1 方式：时间起课 / 干支起课

### 定义

- **时间起课**：输入公历年月日时，系统自动推出四柱干支，再用四柱起大六壬全盘。
- **干支起课**：直接输入年柱、月柱、日柱、时柱，不经由公历时间换算。

### 作用

这个选项决定的是**四柱源头输入方式**，不是盘式流派。

### 影响范围

它会影响：

1. 年月日时干支
2. 所属六甲旬
3. 旬空
4. 月将判断所依赖的时间上下文
5. 之后全部四课、三传、神将、神煞

### 开发要求

- 时间起课必须能够稳定推出四柱
- 干支起课必须允许直接输入完整四柱
- 两者一旦落到同一组四柱，后续起盘结果必须完全一致

---

## 4.2 生年

### 定义

生年不是日常时事占的必要参数，而是**本命占 / 终身占**的扩展参数。

### 两种用途必须分开

1. **时事占**
   问求职、感情、财运、谋事、吉凶时，只以占时的日时四柱起课，生年可以为空。
2. **本命终身占**
   需要把出生年、本命宫、行年等附加信息并入解盘时，生年必须接入。

### 影响范围

生年不影响基础四课、三传、月将、贵神。

它影响的是：

- 本命宫位
- 行年
- 终身运分析
- 与求测人本命相关的附加判断

### 开发要求

- 时事占流程中，生年必须是可选字段
- 本命占启用前，不得把生年误接入基础排盘主链

---

## 4.3 月将

### 定义

月将是大六壬起盘的第一核心参数。

六壬铁则：

> 月将加时，方起天地盘。

整个盘里以下内容，都以月将为根源：

1. 天盘十二支
2. 十二天将落宫
3. 四课上神
4. 三传中后续的阴神链
5. 神煞落宫解释

### 正统规则

月将按**中气后的太阳过宫**确定，不是直接拿月支当地将。

| 中气之后 | 月将 |
|---|---|
| 雨水后 | 亥 |
| 春分后 | 戌 |
| 谷雨后 | 酉 |
| 小满后 | 申 |
| 夏至后 | 未 |
| 大暑后 | 午 |
| 处暑后 | 巳 |
| 秋分后 | 辰 |
| 霜降后 | 卯 |
| 小雪后 | 寅 |
| 冬至后 | 丑 |
| 大寒后 | 子 |

### 为什么界面必须允许“自动 / 手动”

因为现实中存在两类使用方式：

1. **系统自动选将**
   这是正统默认模式，按中气自动匹配太阳月将。
2. **手动指定月将**
   用于兼容少量旧传本、教学演示或人工校盘。

### 对盘内结构的影响

月将一改，下面这些全部整体偏移：

1. 天盘 12 宫映射
2. 四课四个上神
3. 三传中依赖天盘的阴神链
4. 十二天将落宫
5. 神煞落宫解释

### 开发要求

- 默认必须是系统自动选将
- 自动选将必须以中气为准
- 手动月将只能作为显式 override，不能取代自动逻辑

### 选错后果

月将一错，不是“某一宫错一点”，而是**全盘错盘**。

最典型的错误，就是把“节气边界附近的真实月将”退化成“按月支粗暴映射月将”，这样会直接导致：

- 四课全变
- 三传全变
- 神将全变
- 神煞落宫解释失效

---

## 4.4 昼夜：昼贵 / 夜贵 / 自动

### 定义

昼夜选项用于判断本课使用**昼贵**还是**夜贵**。

### 古法规则

- 卯时至酉时：昼
- 酉时至次日卯时：夜

### 作用

它决定：

1. 当前课取昼贵还是夜贵
2. 贵人宫位
3. 十二天将排布方向

### 为什么它极重要

大六壬不是“只换一个贵人字”。

一旦昼夜切换，整个十二天将的落宫顺序都会跟着变。

### 对盘内结构的影响

昼夜一改，下面这些全部会变化：

1. 贵人落宫
2. 天将顺布 / 逆布方向
3. 十二天将在各宫的对应关系
4. 由天将参与解释的四课、三传象意

### 开发要求

- 默认必须是自动
- 自动必须按时辰判断昼夜
- 昼贵 / 夜贵强制项必须保留，方便教学和人工校盘

### 选错后果

昼夜选错，贵人位置错，天将排布方向错，整套神将系统全部错位。

---

## 4.5 贵人口诀版本

### 定义

这个选项不是装饰项，而是处理不同传本口诀差异的兼容项。

当前必须支持的基准版本：

- `甲戊庚牛羊`

可兼容版本：

- `甲羊戊庚牛`

### 规则说明

两套口诀差异，主要影响甲日的贵人定位。

对壬日来说，两套口诀不会改动贵人位置。

### 对盘内结构的影响

一旦口诀切换并且命中了受影响日干，会引发：

1. 贵人宫位改变
2. 整个十二天将重新排布
3. 四课、三传上的天将注记改变

### 开发要求

- 默认固定使用 `甲戊庚牛羊`
- 若要开放其它版本，必须显式标明是兼容选项
- 不能在后台悄悄切口诀

---

## 4.6 三传旬位：日柱旬遁干 / 时柱旬遁干

### 定义

这个选项决定整套遁干、旬空、取传主轴以哪个柱为准。

### 正统默认

大六壬日常占事，正统默认必须是：

- **日柱旬遁干**

### 原因

因为大六壬的主轴是：

- 日干为我
- 日支为事

所以旬空、遁干、发用判断，都应围绕日柱展开。

### 时柱旬遁干是什么

这是偏门或支流用法。

一旦切成时柱旬遁干，会直接改变：

1. 所属六甲旬
2. 旬空
3. 遁干
4. 发用判断背景

### 对盘内结构的影响

这不是“只换空亡两个字”，而是会连锁影响：

1. 空亡标识
2. 四课对空亡的解释
3. 三传取用背景
4. 部分神煞解释

### 开发要求

- 默认必须是 `日柱旬遁干`
- 若开放 `时柱旬遁干`，必须明确标为非默认兼容模式

---

## 4.7 三传显示在上

### 定义

这是结果页排版开关，不改变盘义。

### 作用

只影响：

- 三传显示在最上方还是结果区块中部

### 不影响内容

它不影响：

- 四课
- 三传地支
- 月将
- 贵神
- 神将
- 神煞

### 开发要求

- 必须明确标注这是“显示选项”，不是“算法选项”

## 5. 大六壬正统排盘链路

大六壬正统排盘，必须按以下顺序实现，不能跳步骤，也不能先做显示再反推结构。

### 5.1 取得源头四柱

来源可以是：

- 公历时间推四柱
- 手动输入完整四柱

无论哪种入口，只要四柱相同，后续排盘结果必须相同。

### 5.2 定旬空

先确定本课属于哪一旬，再确定旬空。

这一步决定的是：

- 地支空亡
- 遁干背景
- 后续解释中的虚实基础

### 5.3 定月将

月将必须按中气后的太阳过宫确定。

不得用“当前农历月支”直接代替月将。

### 5.4 定昼夜

先判时辰属于昼还是夜，再决定本课用昼贵还是夜贵。

不得把“日干阴阳”混进这一步，导致昼夜判断和布将方向杂糅。

### 5.5 定贵人

先根据日干和口诀版本确定贵人候选，再根据昼夜取昼贵或夜贵。

不得出现：

- 界面显示“夜贵”，实际却取了昼贵宫位
- 界面显示“昼贵”，实际却按夜课逆布

### 5.6 月将加时，起天地盘

公式必须是：

> 月将加在占时上，顺布十二地支。

这一步的直接输出必须至少包含：

1. 地盘 12 支
2. 各地盘宫对应的天盘 12 支

### 5.7 排十二天将

必须先有：

1. 贵人落宫
2. 当前课是昼还是夜

再决定十二天将顺布还是逆布。

输出必须至少包含：

- 12 将名称
- 各将所临地支
- 各将所乘天盘

### 5.8 排四课

四课属于正统盘的核心结构，不能只存一个“上神 / 下神 / 天将”简表就算完成。

至少必须表达：

1. 第一课、第二课、第三课、第四课的古法顺序
2. 每课的上神
3. 每课的下神
4. 每课的天将
5. 每课的六亲
6. 每课的上下生克关系

### 5.9 取三传

三传不能简单理解为“从天盘 map 连取三次”。

正统实现至少要有：

1. 发用判定
2. 课体名称
3. 取传说明
4. 初传、中传、末传
5. 每传六亲
6. 每传天将

### 5.10 叠加神煞、六亲、遁干、空亡

这一步是辅助层，不是主链起点。

必须建立在前面的天地盘、四课、三传已正确的前提上。

## 6. 固定样例：2026年4月18日 22:20

### 6.1 为什么用这个样例

这个时间已经被反复用于“正统盘”和“错误盘”的对比，适合作为标准 worked example。

说明：

- 若使用 `2026年4月18日 22:12`，同样属于辛亥时
- 在本样例涉及的月将、昼夜、贵神、四柱规则上，结论相同

本文统一用：

- **2026年4月18日 22:20**

### 6.2 基础四柱

该时刻对应：

- 丙午年
- 壬辰月
- 壬戌日
- 辛亥时

旬空：

- 壬戌日属甲寅旬
- 甲寅旬子丑空

月建与日建：

- 月建辰
- 日建戌

### 6.3 参数面板的标准选择

此样例的正统参数应固定为：

| 参数 | 正确选择 |
|---|---|
| 方式 | 时间起课 |
| 生年 | 空，不参与时事占 |
| 月将 | 系统自动选将 |
| 昼夜 | 自动 |
| 贵人口诀 | `甲戊庚牛羊` |
| 三传旬位 | 日柱旬遁干 |
| 三传显示在上 | 勾选 |

### 6.4 月将的校盘结论

本样例位于：

- 春分之后
- 谷雨之前

对应正统月将应为：

- **戌将**

原因：

- 六壬月将按中气后的太阳过宫确定
- 本时刻尚未进入谷雨中气
- 因此不能提前使用谷雨后的酉将

### 6.5 昼夜、贵人、布将方向的校盘结论

本样例时支为：

- 亥时

亥时属于：

- 夜课

日干为：

- 壬

按 `壬癸兔蛇藏`：

- 壬日昼贵在巳
- 壬日夜贵在卯

因此本样例的正统结论是：

- **夜贵在卯**
- **十二天将按夜课逆布**

### 6.6 样例中的十二天将校盘基线

本样例用于结果页和单元测试时，至少应能人工核对出以下方格关系：

```text
蛇 雀 合 勾
辰 巳 午 未

贵 卯        申 龙
后 寅        酉 空

阴 玄 常 虎
丑 子 亥 戌
```

说明：

- 这是样例里的十二天将展开基线
- 结果页可以用卡片、方盘、九宫式排版表达
- 但不得只剩“贵人在卯，夜课逆布”两行摘要

### 6.7 样例中的三传校盘基线

本样例的结构化三传基线为：

| 传 | 地支 | 六亲 | 天将 |
|---|---|---|---|
| 初传 | 戌 | 官鬼 | 虎 |
| 中传 | 酉 | 父母 | 空 |
| 末传 | 申 | 父母 | 龙 |

开发要求：

- 结果页必须能直观看到初中末三传
- 必须能看到三传对应的六亲和天将
- 必须能看到课体名称与取传说明

### 6.8 样例中的四课校盘基线

本样例对应的四课，必须同时保留两种表达：

1. 古法原序
2. 现代显示序

#### 古法原序

古法原序是：

- 第一课在最右
- 第四课在最左

本样例的古法原序基线如下：

| 课序 | 天将 | 上神 | 下神 |
|---|---|---|---|
| 第一课 | 虎 | 戌 | 壬 |
| 第二课 | 空 | 酉 | 戌 |
| 第三课 | 空 | 酉 | 戌 |
| 第四课 | 龙 | 申 | 酉 |

#### 现代镜像显示序

若 UI 采用现代从左到右显示，推荐镜像为：

```text
龙 空 空 虎
申 酉 酉 戌
酉 戌 戌 壬
```

开发要求：

- 数据层必须保留 `lessonIndex = 1..4`
- 显示层可以镜像
- 但取传逻辑永远按古法课序处理，不能按界面视觉顺序处理

### 6.9 样例里的两个高频易混点

#### 旬空和天空不是一回事

必须明确区分两套体系：

- `甲寅旬 子丑空`：这是**旬空**
- 盘面里某宫位标注 `空`：通常是该位乘了**天空天将**

开发要求：

- 结果页不得把旬空和天空混为一个字段
- 样式上可以接近，但数据结构必须分开

#### 四课古法原序和现代显示序不是一回事

古法四课原序是：

- 第一课
- 第二课
- 第三课
- 第四课

并且原生习惯是**从右向左读**。

现代 UI 为了可读性可以镜像成从左向右显示，但必须满足两条：

1. 数据模型里仍保留真实课序
2. 文档和代码里明确标出“显示序 != 古法原序”

## 7. 结果页布局规范

### 7.1 区块顺序

大六壬结果页必须按以下顺序展示：

1. 占问事宜
2. 基础信息
3. 三传
4. 四课
5. 天地盘
6. 十二天将
7. 神煞
8. AI 分析

说明：

- 若开启“三传显示在上”，三传必须位于四课之前
- AI 分析永远在结构化排盘之后

### 7.2 各区块必须显示的要素

#### 占问事宜

- 问题文本
- 如无内容可收起

#### 基础信息

必须显示：

- 阳历时间
- 农历日期
- 年、月、日、时干支
- 旬空
- 月建
- 日建
- 月将
- 昼贵 / 夜贵
- 当前采用的口诀与旬位规则

#### 三传

必须显示：

- 初传、中传、末传
- 每传地支
- 每传六亲
- 每传天将
- 课体名称
- 取传说明

#### 四课

必须显示：

- 第一课、第二课、第三课、第四课
- 每课上神
- 每课下神
- 每课天将
- 每课六亲
- 每课上下生克关系

附加要求：

- 必须明确古法原序
- 必须明确界面显示序
- 若使用镜像布局，不能在数据层把课序重排后再丢失原始顺序

推荐说明文案：

- 古法原序：第一课在最右，第四课在最左
- 现代显示：可左到右展示，但逻辑层仍按第一课到第四课处理

#### 天地盘

必须显示完整 12 宫：

- 地盘地支
- 对应天盘地支

不允许只显示“月将摘要”两行代替完整天地盘。

#### 十二天将

必须显示完整 12 将：

- 贵人
- 腾蛇
- 朱雀
- 六合
- 勾陈
- 青龙
- 天空
- 白虎
- 太常
- 玄武
- 太阴
- 天后

每一将至少要显示：

- 神将名
- 所临地支
- 所乘天盘

不允许只显示“贵人位置 + 布神方向”两行。

#### 神煞

必须区分：

- 吉神
- 凶神

每条至少显示名称与所落宫位。

#### 解读层边界

结构化排盘和断语解释必须分层。

本规范约束的是：

- 干支
- 月将
- 贵神
- 四课
- 三传
- 天地盘
- 神将
- 神煞

本规范不约束：

- 人工断语文案
- AI 解读风格
- 不同门派的象意展开

原因：

- 结构错了，后面的断语再像也没有意义
- 结构先统一，解读层才能稳定演化

## 9. 当前代码与本规范的偏差

这一节是给开发者看的，不是给用户看的宣传文案。

当前代码存在以下结构性偏差，后续重写必须优先解决。

### 9.1 月将链路不可靠

当前代码依赖 `lunar.getJieQi()`。

当库返回空字符串时，会退回“按月建映射月将”。

这会把“中气判月将”退化成“月支映射月将”，直接导致节气边界附近起错月将。

### 9.2 贵人与布神方向规则未按本文定义收敛

当前实现把：

- 昼夜贵
- 日干阴阳

混在一起决定布神顺逆。

后续需要按本文的界面参数与规则体系重新拆分：

1. 先定昼夜贵
2. 再定采用哪套布将规则
3. 把规则作为显式参数而不是服务内隐常量

### 9.3 四课模型仍然过于简化

当前 `Ke` 只保存：

- 上神地支
- 下神地支
- 天将

这不够表达正统大六壬的完整四课结构。

后续至少要补：

- 更明确的本位定义
- 六亲
- 上下生克关系
- 古法顺序与界面顺序的双重表示
- 为遁干扩展预留结构

### 9.4 三传当前仍是极简链式占位算法

当前实现里的中传、末传，本质上还是沿天盘 map 连取，不足以表达完整取传链。

后续必须把以下内容显式化：

1. 发用来源
2. 课体名称
3. 初传判定依据
4. 中传、末传推导依据

### 9.5 结果页缺少完整天地盘和十二天将

当前 UI 只展示：

- 月将摘要
- 贵人摘要
- 布神方向摘要

这不能满足人工校盘，也不能满足正统排盘使用。

### 9.6 当前样例会稳定产出错误盘

对固定样例 `2026-04-18 22:20`，当前代码会产出如下错误结果：

- `月将 = 酉`
- `贵人 = 巳`
- `direction = 阳日顺布`
- 四课：`酉/亥`、`未/酉`、`申/戌`、`午/申`
- 三传：`午 -> 辰 -> 寅`
- `keType = 比用`

而本文档规定的正统校盘基线是：

- `月将 = 戌`
- `夜贵 = 卯`
- `夜课逆布`
- 三传基线：`戌 -> 酉 -> 申`

因此后续重写时，不允许以“兼容旧结果”为理由保留错误逻辑分支。

## 11. 旧规则处理原则

本系统后续重构时，不需要为了“兼容旧错误盘”保留错误解析逻辑。

不可兼容的包括：

## 13. 结构化文字输出规范

大六壬除了结果页和历史卡片，还必须有一份**稳定、可程序生成、可供 AI 直接消费**的文字版排盘格式。

这份格式不是自由文案，而是正式契约。  
`lib/ai/output/formatters/daliuren_formatter.dart` 的输出必须与本节一致。

### 13.1 标题与区块顺序

结构化文字输出的标题固定为：

```text
【大六壬完整结构化排盘】
```

其后区块顺序固定为：

1. 排盘总览
2. 天地盘全宫（地盘→天盘）
3. 十二天将完整分布
4. 四课（天盘/地盘/天将/生克）
5. 三传
6. 神煞
